# Azure Storybook — Copilot Instructions

This project is a **Storybook** showcasing **Fluent UI React v9** components themed as **Azure Portal**.

## Tech Stack
- React 18 + TypeScript 5.7
- Fluent UI React v9 (`@fluentui/react-components`)
- Storybook 8.5 (CSF3 format)
- Vite 6 bundler
- Griffel CSS-in-JS (idiomatic to Fluent v9)

## Theme System
- Azure themes live in `src/themes/azureThemes.ts`
- Three themes: `azureLightTheme`, `azureDarkTheme`, `azureHighContrastTheme`
- Brand ramp based on Azure brand blue (#0f6cbd primary)
- Token reference: `src/themes/coherenceTokens.ts`
- Storybook wraps all stories in `<FluentProvider>` with theme switching

## Component Taxonomy
| Sidebar Category | Contents |
|---|---|
| Foundations | Colors, Typography, Spacing, Shadows — visual token reference |
| Components | Stock Fluent v9: Button, Input, Card, Dialog, DataGrid, DataDisplay, Selection |
| Composed | Azure-specific compositions: AzureBreadcrumb, PageTitleBar, CommandBar, FilterBar |
| Templates | Full page examples: ResourceListPage |

## Story Format (CSF3)
```tsx
import type { Meta, StoryObj } from '@storybook/react';

const meta: Meta<typeof Component> = {
  title: 'Category/ComponentName',
  component: Component,
  tags: ['autodocs'],
};

export default meta;
type Story = StoryObj<typeof Component>;

export const Default: Story = {
  args: { /* props */ },
};
```

## Naming Convention — LLM-Friendly, Descriptive Names
- **Name everything by what it does, not where it's used.** Names should describe the *functionality or behaviour* so any reader (human or LLM) can instantly tell what it is and whether to use it.
- Story exports: Use pattern-based names like `WithDivider`, `WithOverflow`, `WithFarItems` — NOT location-specific names like `ResourceList`, `CostManagement`.
- Components & props: Name by capability — `FilterBar`, `CommandBarGroup`, `farItems` — NOT by a single use case like `ResourceFilterBar`.
- Files & folders: Use generic functional names (`CommandBar.stories.tsx`) — NOT context-specific ones (`CostManagementCommandBar.stories.tsx`).
- CSS class names: Describe structure — `leftSection`, `bladeContent`, `gridCell` — NOT domain — `vmList`, `subscriptionPanel`.
- This rule ensures stories and components are **reusable across any Azure page** and that an LLM scanning the registry can quickly match a component to a need without domain-specific knowledge.

## Code Conventions
- Use `makeStyles` from `@fluentui/react-components` for styling (Griffel)
- Reference `tokens` from `@fluentui/react-components` for theme-aware values
- Import icons from `@fluentui/react-icons` (24px Regular variants by default)
- Composed components live in `src/components/`
- Stories live in `src/stories/{category}/`

## Registry
- `src/component-registry.json` — auto-generated list of all components + stories
- Run `npm run build:registry` to regenerate

## Icons
- **ALWAYS** use the `iconcloud-browser` skill (`.github/skills/iconcloud-browser/SKILL.md`) when searching for, downloading, or referencing any icon — Azure service icons, Fluent icons, or any other Microsoft icon.
- **NEVER** guess icon names or hardcode icon references without first verifying them through IconCloud.
- If an icon cannot be found via IconCloud, list it explicitly in your response under a **"Icons Not Found"** section so the user can resolve them manually.

## Azure Context
- Colors derived from Azure Portal's Coherence design system
- Coherence uses `cui-` prefixed web components
- This Storybook recreates those patterns with Fluent v9 React components
- The full Coherence component manifest is documented in `src/themes/coherenceTokens.ts`
