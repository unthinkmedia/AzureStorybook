## Adding a New Story

Every story uses CSF3 format:

```tsx
import type { Meta, StoryObj } from '@storybook/react';

const meta: Meta<typeof Component> = {
  title: 'Category/ComponentName',
  component: Component,
  tags: ['autodocs'],
};

export default meta;
type Story = StoryObj<typeof Component>;

export const Default: Story = {
  args: { /* default props */ },
};
```

## Naming Conventions

Names should describe **what a thing does**, not where it's used. This keeps the library generic and reusable.

| Layer | ‚úÖ Good | ‚ùå Bad |
|---|---|---|
| Story export | `WithDivider`, `WithOverflow` | `ResourceList`, `CostView` |
| Component | `FilterBar`, `CommandBar` | `ResourceFilterBar` |
| CSS class | `leftSection`, `gridCell` | `vmList`, `subscriptionPanel` |
| File name | `CommandBar.stories.tsx` | `CostManagementBar.stories.tsx` |

## Regenerating the Registry

After adding new stories or components, regenerate the component registry:

```bash
npm run build:registry
```

This updates `src/component-registry.json`, which provides machine-readable metadata for LLM context.
